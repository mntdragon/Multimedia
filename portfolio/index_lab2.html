RWD

Definition

Responsive web design (RWD) is one of the trendiest technology buzz words in web development. The terms was originally introduced in Ethan Marcotte’s A List Apart article.



What is the purpose of the RWD? Why it is used? How things are done before RWD? What is the difference?

Responsive web design is likely inheritance of Responsive architecture. Responsive Web is a unique web version can run on all devices such as PCs, smartphones, or tablets. There are more and more devices these days, using website on portable devices process is growing extremely quick, the demand for a mobile version of the website is growing in parallel. The RWD has met the needs for that demand which satisfy many sizes interfaces on different devices. It displays the fullscreen without broken interface or some elements in some positions are hidden or no need to zoom in the screen to see on any devices.
Before RWD, people have to build different versions for different devices with different screen sizes. Therefore, there would be different URL and HTML for mobile devices, it would not be easy and convenient for people to share the links and interact with the distinct sites for mobile. The old version website, images would be easiy distortion and break the structure of the site, also the speed of loading page would be pretty slow. Moreover, with RWD, people can save more time and cost for developing website due to its responsiveness and optimization. Stable quality website and easily adaptable to user needs. This helps website become more professional and have the opportunity to interact with a broader audience.



How css media queries can help you building responsive web sites?

Media queries are the key to help us be able to write CSS for responsive websites wich is a new techniques in CSS3.

@media not|only mediatype and (media feature) {CSS-Code;}

In here, media types are print, screen and all. Media features are min-width, min-height, aspect-ratio, orientation, color-index and etc. CSS media queries has princible which is used through the screen size parameters declared via @media. With this, people can segment their css into many different parts corresponding to the size of the device types. For instance, we have CSS code for iPhone 4:
@media only screen
and (min-device-width: 320px)
and (max-device-width: 480px)
and (-webkit-min-device-pixel-ratio: 2) {}

With min-device-width, max-device-width are the min and max width of device. -webkit-min-device-pixel-ratio minimum ratio between the width and height of the device in pixel for Safari.



Why to use em (font) based sizing for example margins?

1em equal to the current font size, 2em = 2 times the current font size. Em is a relative measrure in website as percentage which based on the value of the font-size property. 'Em' is a very useful unit in CSS, especially in the case of users who use zoom to enlarge the text in the browser, the units will stretch out in proportion of the letter. Em is typically used for the elasticity content while zooming. For example margins, to generate space around elements, margin should be optimized and responsive to the website screen, if we use em, element betweens margins would always be symmetric, not cramped anymore, ensure scalable design. It is better to control when people resize the screen.
