Definitions

HTML

HyperText Markup Language, which is used for define structure,format layout and related attributes shown on the brownser. It is as the backbone of the website. Not a programming language. Using tag annd attributes to control form and content of website page, insert objects as video, image, audio clip, etc into HTML, publish and access online sources, documents or info by using inserted links in HTML documents. HTML document has 3 basic parts: html, head and body tag. HTML uses a set of tags to describe a web page. Each tag describre different info.



CSS

Cascading Style Sheets, which is used to format a certain part of HTML, help the code of website be more compact and easy to edit. A language for decorating content of website. CSS can change default attributes of html tags. CSS helps to change the style of a Web page simpler and less time consuming. When HTML has CSS as person has beautiful clothes.



Javascript

A programming language, embedded or insert directly in HTMl file, that help website be more interactive, responsive, vivid. It can be also embedded in Java. While making website,people usually use a library source for Js as jQuery fo rexample. Js language has many features: object oriented, simple, ... One of the most important feature is the creative ability to create and use objects to develop applications more easily. Advantages of Javascript are: less server interaction, response quickly to customer or user, increasing the interactive ability, plentiful interfaces and templates, checking data validity, making and changing dynamic content.



HTTP

Hypertext Transfer Protocol, a way to transmit data through internet, while loading website, using the HTTP protocol to send and receive data. HTTP can be seen everywhere. HTTP works based on the Client-Server model. HTTP helps enable the transmission of files from a Web server to a Web browser so users can view a Web page presenting in web brownser.



HTML5

A new better version of HTML, displays for WWW. HTML5 has added new features to build web applications much easier. Example: more color and support curve, rounded corner, ... HTML5 will allow you to watch videos without using plugin like Flash or Silverlight. With HTML5, website becomes more visible, more attractive and make everything becomes closer to what the designer imagined in mind.



DHTML

A combination of HTML, CSS, DOM and Javascript. It is aas an ecpanded version of HTML, to increase presenting content effects and other objects. DHTML's structure has 4 parts: DOM, Script, CSS, XHTML. DHTML makes website dynamic. For instances, paragraph can be changed posion, color or style, respond well the impact of user to website. There are some features in DHTML: change tag and property(most common feature), real-time positioning, dynamic fonts and data binding.

Coding conventions

A collection of standard rules for coding in a project. This makes long codes are clear,durable, consistent, flexible and easy to read, understand, maintain and manage.Especially for working in a team, coding conventions help the process running smoothly. Each member can use the products of other members when they have detailed design documents. Team would easily predict or understand your code and fit the others well. This could save time-consumong not only for team but also for your own.



Some kinds of conventions which are applied while I am writing codes:

- Naming convention: camelCase(productPrice, animalSmall) which using lowecase for the first letter of the word and uppercase for the first letter of the next word; PascalCase(ProductPrice) which using uppercase eery first letter of each word; SnakeCase(product_price, student_score) which using lowercase for all letter, and words separated by underscores. In general, when puting name, I usually use meaning noun or verb, avoid using the acronym and non-meaning or too common name.
- Make sure every tag has their closing one, even the optional tag.
- Use quotation for attribute values
- Use lowercase for attribute names: <section> is much better than <Section> or <SECTION>
- Use a space before and after the operator for the clear and readibility code. Except for unary operators as ++ or--. For example: this.barName = "Bonny yeu Pham Huong";
- Always add comments well to describe code more understandable and approachable, help I and other who read my code understand and explore more quickly. Moreover, comments is pretty good to fix and improve code. They should be written with descriptive info and what the program should be noted for reading and editing code later.
- For style definitions, using empty line between them and a space after semicolons to make the code professional and readable.
- Using alt attribute every time inserting image. This convention make sure the engine searchs and understand what the image would like to say, its aim, especially for the not showing image.

Developer Tools

For web developers, the web browser is the most exposed for them after only IDE. In the web brownser, there is a hidden tool which is very usefull for developer check the HTML, CSS code or debug javascript code. Right-click on any website, click Inspect and you can see everything inside of that website, including source code, images and CSS design, fonts and icons and JavaScript code animations, ... I usually use Google Crome and its Developing tools for studying and exploring the codes of the websites. This help saving time and improve my working efficiency.

How to open the Dev tool: To start you open Google Chrome, and then press the F12 key to launch it or right click, then click Inspect. From the Developer Tools window, you will see these panels:

  • Elements
  • Resources
  • Network
  • Source
  • Timeline
  • Profile
  • Audits
  • Console

You can use Ctrl + [or Ctrl +] to move between tabs controls.

In Console panel you can find errors of javascript. The errors related to JavaScript (not loaded, missing closing parenthesis opened, semicolon, ...) will be displayed in this panel. You can also run the javascript code directly on this panel. In the panel(or tab) Source, you can put breakpoint while debugging. You can both call debugger function in code or click the number line of code in Source panel to mark the breakpoint. Moreover, you can also open the minify Javascript to reduce the transmiting size and increase website speed. By clicking the 'pretty print' in the left bottom corner, all JS code will be turned into minify and return to easy reading format, easy debugging and putting breakpoint.
